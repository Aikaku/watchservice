### 3. Gemini guidance 작동 확인 및 버그 수정

Gemini API 정상 작동 여부를 확인하는 과정에서 3가지 문제 발견 및 수정.

#### 3-1. 테스트 엔드포인트 추가

XGBoost 서버 없이 Gemini만 단독 테스트할 수 있는 엔드포인트 추가.

- **파일**: `ai/AiController.java`
- **엔드포인트**: `GET /ai/gemini/test` — DANGER(LockBit) 시나리오 고정 호출

#### 3-2. bootRun 환경변수 미전달 문제

**원인**: `./start.sh`에서 `.env`를 로드해도 Gradle `bootRun`이 셸 환경변수를 Spring Boot JVM에 전달하지 않음.

**수정**: `build.gradle`에 `bootRun { environment System.getenv() }` 추가.

- **파일**: `build.gradle`

#### 3-3. Gemini API 타임아웃

**원인**: 기본 타임아웃 2500ms가 Gemini 응답 시간(~2.7초)보다 짧아 Read timed out 발생.

**수정**: `application.yml` 기본값 2500 → 10000ms.

- **파일**: `src/main/resources/application.yml`
- **변경**: `gemini.timeout-ms: ${GEMINI_TIMEOUT_MS:10000}`

#### 3-4. Gemini 2.5 Flash thinking 토큰 문제

**원인**: `maxOutputTokens=400`으로 thinking 토큰이 출력 토큰을 소비하여 실제 텍스트가 수 줄에서 잘림.

**수정**:
- `maxOutputTokens` 400 → 1500
- `generationConfig.thinkingConfig.thinkingBudget: 0` 추가 (thinking 비활성화)

- **파일**: `ai/GeminiAdviceService.java`

#### 3-5. start.sh 데몬 재사용 방지

**수정**: `./gradlew bootRun` → `./gradlew bootRun --no-daemon` (키 없이 시작된 기존 데몬 재사용 방지)

- **파일**: `watchservice_be/start.sh`
