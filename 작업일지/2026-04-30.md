## 1. Electron 앱 "서비스 시작 실패" 오류 수정

### 기존 상태
```javascript
// main.js waitForSpring()
if (res.statusCode === 200) {
  resolve();
}
```
```yaml
# application.yml — management 설정 없음
```

### 문제점
Spring Boot의 `MailHealthIndicator`가 SMTP 미설정 상태에서 `/actuator/health`에 503을 반환함.  
`waitForSpring()`은 200만 허용해 60초 타임아웃 후 "서비스 시작 실패" 팝업이 뜸.

### 변경 내용
```yaml
# application.yml
management:
  health:
    mail:
      enabled: false
```
```javascript
// main.js
if (res.statusCode === 200 || res.statusCode === 503) {
  resolve();
}
```
mail health indicator를 비활성화해 503 자체를 없앰. waitForSpring()은 503도 허용하도록 방어 처리.

## 2. 흰 화면(White Screen) 수정

### 기존 상태
```java
// WebConfig.java
registry.addViewController("/{path:[^\\.]*}/**").setViewName("forward:/index.html");
```

### 문제점
`/{path:[^\\.]*}/**` 패턴이 `/static/js/main.abc123.js`의 첫 세그먼트 `static`(점 없음)을 캐치함.  
React 번들 요청이 `index.html`로 포워딩되어 JS가 실행되지 않고 빈 화면이 표시됨.

### 변경 내용
```java
// WebConfig.java — 명시적 React 라우트 목록으로 교체
registry.addViewController("/{path:[^\\.]*}").setViewName("forward:/index.html");
registry.addViewController("/notifications/**").setViewName("forward:/index.html");
registry.addViewController("/logs/**").setViewName("forward:/index.html");
registry.addViewController("/settings/**").setViewName("forward:/index.html");
registry.addViewController("/notice/**").setViewName("forward:/index.html");
registry.addViewController("/admin/**").setViewName("forward:/index.html");
```
와일드카드 패턴 대신 React 라우트를 명시적으로 나열해 `/static/**` 충돌 방지.

## 3. 메인보드 "폴더 추가" 버튼 무반응 수정

### 기존 상태
```javascript
// MainBoardPage.jsx
const { folders, promptAndAddFolder, removeFolder } = useWatchedFolders();
// ...
<button onClick={promptAndAddFolder}>폴더 추가</button>
```

### 문제점
`useWatchedFolders` 훅은 `promptAndAddFolder`를 export하지 않음. undefined를 호출하는 TypeError가 조용히 발생해 모달이 안 열림.

### 변경 내용
```javascript
// MainBoardPage.jsx
import FolderPickerModal from '../../components/folders/FolderPickerModal';
const { folders, addFolder, removeFolder } = useWatchedFolders();
const [showFolderPicker, setShowFolderPicker] = useState(false);

const handleAddFolder = () => setShowFolderPicker(true);

{showFolderPicker && (
  <FolderPickerModal
    onSelect={async (path) => { setShowFolderPicker(false); await addFolder(path, ''); }}
    onClose={() => setShowFolderPicker(false)}
  />
)}
```
실제로 export되는 `addFolder`를 사용하고, `FolderPickerModal`로 직접 상태 관리.
