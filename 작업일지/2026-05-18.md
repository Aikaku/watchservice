## 1. macOS DMG 서명 문제 수정 및 재릴리스

### 기존 상태
`electron-builder` 기본 adhoc 서명: `Sealed Resources=none`

### 문제점
다운로드 후 macOS Gatekeeper가 "앱이 수정 또는 손상되었습니다" 오류 표시. `Sealed Resources=none` 상태에서는 우클릭 → 열기 우회도 불가.

### 변경 내용
`afterSign.js` 훅 추가 → `codesign --deep --force --sign -` 전체 번들 재서명. `package.json`에 `"afterSign": "./afterSign.js"` 등록. GitHub Releases에 재업로드. 다운로드 사이트 URL 수정(대시 → 점), macOS 첫 실행 안내 문구도 "우클릭 → 열기"로 업데이트.

## 3. 인수테스트 결과 기반 버그 수정 (4건)

### 이슈 1: 행위 탐지 후 대시보드 미업데이트

#### 기존 상태
```js
useEffect(() => {
  loadSummary(); // 마운트 1회만 호출
}, []);
```

#### 문제점
탐지 발생 시 새로고침 없이는 상태(DANGER/WARNING/SAFE)와 최근 이벤트가 갱신되지 않음.

#### 변경 내용
```js
// loadSummary를 useCallback으로 추출
const loadSummary = useCallback(async () => { ... }, []);

// 마운트 + 5초 폴링
useEffect(() => {
  loadSummary();
  getWatcherStatus().then(data => { if (data?.running) setIsWatching(true); }).catch(()=>{});
}, [loadSummary]);

useEffect(() => {
  const id = setInterval(() => { loadSummary(); refresh(); }, 5000);
  return () => clearInterval(id);
}, [loadSummary, refresh]);
```
`useLogs(5)`에서 `refresh` 추가 destructure.

### 이슈 3: 긴급 전체화면 경보 확인 버튼 작동 안 함

#### 기존 상태
```html
<button onclick="window.close()">확인</button>
```
`emergencyWindow`에 preload 없음.

#### 문제점
Electron에서 `contextIsolation: true` + preload 없는 창의 `window.close()`는 차단됨.

#### 변경 내용
- `preload.js`에 `closeEmergencyWindow: () => ipcRenderer.send('close-emergency-window')` 추가
- `main.js`: `emergencyWindow`에 `preload.js` 연결 + `ipcMain.on('close-emergency-window', ...)` 핸들러 추가
- `emergency.html`: `window.electronAPI.closeEmergencyWindow()` 호출로 교체

### 이슈 4: 감시/검사 시작 후 다른 페이지 이동 시 버튼 상태 초기화

#### 기존 상태
`isWatching`이 React 로컬 state(`useState(false)`)로만 관리됨.

#### 문제점
페이지를 벗어났다가 대시보드로 돌아오면 컴포넌트가 재마운트되어 `isWatching = false` 초기화 → "감시 시작" 버튼이 다시 표시됨.

#### 변경 내용
백엔드 `GET /watcher/status` 엔드포인트 추가 (`WatcherController.java`).
```java
@GetMapping("/status")
public ResponseEntity<Map<String, Boolean>> status(HttpSession session) {
    return ResponseEntity.ok(Map.of("running", watcherService.isRunning(ownerKey)));
}
```
프론트 `WatcherApi.js`에 `getWatcherStatus()` 추가. 마운트 시 상태 조회 후 `setIsWatching(true)` 복원.

### 이슈 5: AI 대응 가이드 동일한 답변

#### 기존 상태
`fallbackGuidance()`가 인자 없이 고정 문자열 반환.

#### 문제점
`GEMINI_API_KEY` 미설정 시 항상 동일한 텍스트 출력. 탐지 내용과 무관한 범용 체크리스트.

#### 변경 내용
`fallbackGuidance(AiPayload, AiResult)`로 시그니처 변경. 탐지 등급·AI 점수·영향 파일 수·감지 활동(암호화·이름변경 등)을 헤더에 포함하고, 암호화/대량 이름변경 감지 시 해당 조치 항목 우선 표시. 비고란에 Gemini API 키 미설정 안내 추가.

## 이슈 2: 이메일 알림 작동 안 함 (코드 변경 없음)

`SMTP_USERNAME` / `SMTP_PASSWORD` 환경변수 미설정이 원인. `EmailNotificationService`는 해당 값이 비어 있으면 의도적으로 발송을 건너뜀(정상 동작). 이메일 기능 활성화를 위해서는 서버 실행 전 환경변수 설정 필요:
```
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_USERNAME=your@gmail.com
SMTP_PASSWORD=앱비밀번호
```

## 4. macOS 재빌드 및 릴리스 배포

### 기존 상태
GitHub Releases v1.0.0에 이전 빌드(`WatchService.Agent-1.0.0-arm64.dmg`) 업로드된 상태.

### 문제점
`./build.sh --electron` 실행 시 `Unsupported class file major version 69` 오류 발생. 시스템 기본 JDK가 Java 25로 변경되어 Gradle Groovy DSL 컴파일러가 Java 25 클래스파일을 파싱 불가.

### 변경 내용
`watchservice_be/gradle.properties`에 `org.gradle.java.home` 설정으로 Gradle 데몬 JVM을 Java 17로 고정.
```properties
org.gradle.java.home=/Users/sanghyeok/Library/Java/JavaVirtualMachines/ms-17.0.16/Contents/Home
```
빌드 후 `gh release upload v1.0.0 WatchService-Agent-1.0.0-arm64.dmg --clobber`로 새 DMG(174MB) 업로드.
`download_site/index.html` macOS 다운로드 URL 파일명 `WatchService.Agent-` → `WatchService-Agent-`, 파일 크기 155MB → 174MB 수정.
