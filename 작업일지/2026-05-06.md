## 1. README.md 프로젝트 현황 반영 업데이트

### 기존 상태
- 제목이 "WatchService" (에이전트 명칭 누락)
- 다운로드 링크가 `download_site/INSTALL_GUIDE.md` (로컬 상대 경로)
- "구현 예정 기능 (13개)" 섹션 — 이미 구현 완료된 기능들을 예정으로 표기

### 문제점
- GitHub Pages 배포 후 실제 다운로드 페이지 URL이 README에 반영되지 않음
- 구현 완료된 기능들이 여전히 "구현 예정"으로 표기되어 현황과 불일치
- 프로젝트 GitHub 링크·배포 링크가 README 상단에 없어 접근성 낮음

### 변경 내용
- 제목 → `WatchService Agent — 랜섬웨어 실시간 탐지 시스템`
- 상단에 배지(badge) 3개 추가: 다운로드(v1.0.0), GitHub 레포, 릴리스
- 방법 A 다운로드 링크 → `https://aikaku.github.io/watchservice/download_site/` 실제 URL 교체
- "구현 예정 기능 (13개)" → "추가 구현 기능"으로 섹션명 변경, 구현된 목록 정리
  - 삭제: "사용자 정의 탐지 규칙", "감시 폴더 삭제·이름 변경 감지" (미구현)
  - 추가: "컴퓨터 시작 시 자동 실행" (설정 페이지 토글)
- 관련 문서 섹션 `구현예정_기능명세.md` 설명 수정

## 2. 문서 폴더 생성 및 파일 정리

### 기존 상태
프로젝트 루트에 md 문서 파일들과 아이콘.png가 혼재.

### 문제점
루트 디렉터리가 지저분하고, 문서 파일과 설정 파일이 구분되지 않음.

### 변경 내용
`문서/` 폴더를 생성하고 다음 파일 이동:
- `구현예정_기능명세.md`
- `유스케이스.md`
- `진행상황표.md`
- `추가기능_후보.md`
- `테스트_가이드.md`
- `배포_가이드.md`
- `아이콘.png`

`README.md` 관련 문서 섹션의 링크 경로를 `문서/` 하위 경로로 일괄 수정.
`테스트_가이드.md`, `배포_가이드.md` 항목도 관련 문서 섹션에 신규 추가.

## 3. 파일 권한 감사 기능 제거 — 프로젝트_소개.md

### 기존 상태
`프로젝트_소개.md` Development 섹션 주요 기능 표에 "파일 권한 감사" 항목 존재.
유스케이스 다이어그램에 UC-AUD01 (파일 권한 보안 감사 실행) 섹션 존재.

파일 권한 감사 기능을 구현하지 않기로 결정. 문서에만 남아 혼란 유발.

### 변경 내용
- Development 주요 기능 표에서 "파일 권한 감사" 행 삭제
- 유스케이스 다이어그램에서 "파일 권한 감사" 섹션(UC-AUD01) 전체 삭제
