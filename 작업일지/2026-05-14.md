## 1. Electron 앱 "서비스 시작 실패" 원인 표시 개선

### 기존 상태
```javascript
// app.whenReady catch
mainWindow.loadFile(path.join(__dirname, 'assets', 'error.html'));

// springProcess.on('exit')
mainWindow.webContents.loadFile(path.join(__dirname, 'assets', 'error.html')).catch(() => {});
```
```html
<!-- error.html -->
<div class="desc">
  백엔드 서버를 시작하는 데 실패했습니다.<br>
  로그를 확인하거나 앱을 재시작해 주세요.
</div>
```

### 문제점
Windows 사용자가 앱 실행 후 7~8초 만에 "서비스 시작 실패" 팝업이 뜨는데, 원인을 알 수 없었음. 7~8초는 JVM이 기동되다가 포트 8080 충돌로 죽는 시간과 일치. 기존 코드는 어떤 이유로 실패했든 동일한 메시지만 표시함.

### 변경 내용

**`watchservice_electron/main.js`**
- `net` 모듈 import 추가
- `springExited` / `springErrorReason` 전역 변수 추가
- `checkPortAvailable(port)` 함수 추가 — TCP bind 시도로 포트 점유 여부 사전 확인
- `showError(reason)` 헬퍼 추가 — `error.html`에 `?reason=` 쿼리 파라미터 전달
- `startSpringBoot()` stderr 핸들러에 "already in use" 감지 추가 → `springErrorReason = 'port'` 저장
- `startSpringBoot()` exit 핸들러: 앱 기동 완료 후 크래시인 경우에만 직접 오류 표시 (기동 중 크래시는 `waitForSpring` reject로 처리)
- `waitForSpring()`: `springExited` 플래그 즉시 감지 → reject('crash'), 타임아웃 reject 메시지 → 'timeout'
- `app.whenReady()`: Spring 시작 전 포트 사전 체크 추가, catch에서 `showError(reason)` 호출

**`watchservice_electron/assets/error.html`**
- `reason` 쿼리 파라미터 파싱하여 원인별 메시지 표시
  - `port`: "포트 8080이 이미 다른 프로그램에서 사용 중입니다."
  - `crash`: "백엔드 서버가 예기치 않게 종료되었습니다."
  - `timeout`: "서버가 60초 내에 응답하지 않았습니다."

## 2. 랜섬웨어 패밀리 상세 모달 기능 제거

### 기존 상태
알림 상세 페이지에서 탑 패밀리 이름을 클릭하면 `FamilyInfoModal`이 열리고 `/api/families/{name}` API를 호출해 공격 방식·주요 타깃·대응 방법 등을 표시하는 기능이 있었음.

### 문제점
AI 모델이 이진 분류기(benign/ransomware)라 `topFamily`에 구체적인 패밀리명(LockBit, Ryuk 등)이 아닌 "ransomware"가 저장됨. `resources/families/ransomware.json`이 없어 클릭 시 항상 "패밀리 정보를 찾을 수 없습니다." 오류가 표시됨.

### 변경 내용
`NotificationDetailPage.jsx`에서 `FamilyInfoModal` import, `familyModalName` state, 클릭 버튼, 모달 렌더링 제거. 탑 패밀리는 일반 텍스트로만 표시.
