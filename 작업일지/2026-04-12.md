## 1. 시스템 아키텍처 문서 작성

### 기존 상태
아키텍처 관련 문서가 `CLAUDE.md`의 간략한 설명 외에는 없었음. 컴포넌트 간 관계, API 목록, 배포 구성 등을 한 곳에서 파악할 수 없었다.

### 문제점
- 신규 기여자나 유지보수 시 코드를 전체 탐색해야만 전체 구조 파악 가능
- AI 서버 엔드포인트, 환경변수, DB 테이블, 인증 흐름 등이 코드에 분산되어 있어 참조 비효율

### 변경 내용
`시스템_아키텍처.md` 신규 작성. 포함 내용:
1. **전체 시스템 구성** — Electron → Spring Boot → Python AI Server → Gemini API → GitHub Releases API 계층도 (ASCII 다이어그램)
2. **핵심 탐지 파이프라인** — WatcherService → FileCollectorService → EventWindowAggregator → AiService → GeminiAdviceService → LogService/NotificationService 단계별 상세 흐름
3. **백엔드 패키지 구조** — 7개 도메인 패키지 전체 클래스/역할 나열
4. **프론트엔드 구조** — api/ hooks/ components/ pages/ 레이어별 정리 + 라우트 전체 목록
5. **인증 구조** — UserAuth(Tier 1, 선택적) + AdminAuth(Tier 2) + ownerKey 격리 설명
6. **Python AI 서버 구조** — 모델 로딩 순서, /api/analyze 판정 로직, /predict 응답 형식
7. **Electron 래퍼 흐름** — app.whenReady → 로딩 창 → Spring Boot 시작 → 헬스체크 → 메인 창 → DANGER 폴링 순서
8. **DB 테이블 개요** — 6개 테이블 컬럼 요약
9. **REST API 엔드포인트 목록** — 일반 사용자 26개 + 관리자 전용 9개
10. **환경변수 목록** — 16개 변수 기본값/설명
11. **배포 구성** — Electron 패키징, AI 서버 로컬/원격(Railway) 배포
12. **주요 설계 결정 사항** — SQLite 선택, 3초 윈도우, 엔트로피 샘플링, 비동기 로그 저장 등 12개 결정 이유
