## 1. Windows 경로 처리 지원

### 기존 상태
```java
// SettingsController.java
File dir = path.isBlank() ? new File(System.getProperty("user.home")) : new File(path);
File[] children = dir.listFiles(f -> f.isDirectory() && !f.getName().startsWith("."));
String parentPath = dir.getParent(); // Windows 드라이브 루트(C:\)에서 null 반환
```

```jsx
// FolderPickerModal.jsx
{parentPath && (   // parentPath=""(빈 문자열)이면 false → 상위 폴더 버튼 미표시
  <button onClick={() => navigate(parentPath)}>← 상위 폴더</button>
)}
```

```java
// FileCollectorService.java
int lastDot = pathStr.lastIndexOf('.');  // 경로 전체에서 dot 검색 → 폴더명 dot 오탐 가능
String ext = pathStr.substring(lastDot + 1);
```

### 문제점
- Windows에서 `path=""`로 요청 시 드라이브 목록(C:\\, D:\\) 대신 홈 폴더가 표시됨
- `C:\` 드라이브 루트에서 `getParent()`가 `null` → 상위 폴더 버튼 비활성, 드라이브 목록으로 복귀 불가
- `C:\user.name\file.txt`처럼 폴더명에 `.`이 있으면 확장자 추출 오탐 가능
- Windows 숨김 폴더(`$Recycle.Bin` 등) 목록에 포함됨

### 변경 내용
**`SettingsController.java`**
```java
boolean isWindows = File.separatorChar == '\\';

// path="" + Windows → 드라이브 목록 반환
if (isWindows && (path == null || path.isBlank())) {
    File[] roots = File.listRoots();
    List<Map<String, String>> driveEntries = new ArrayList<>();
    for (File root : roots) {
        entry.put("name", root.getAbsolutePath());
        entry.put("path", root.getAbsolutePath());
    }
    result.put("current", "");
    result.put("parent", null);
    result.put("entries", driveEntries);
    return ApiResponse.ok(result);
}

// 숨김 폴더 제외
dir.listFiles(f -> f.isDirectory() && !f.isHidden() && !f.getName().startsWith("."));

// 드라이브 루트에서 parent=null → "" 센티넬(드라이브 목록 복귀)
String parentPath = dir.getParent();
if (parentPath == null && isWindows) parentPath = "";
```

**`FolderPickerModal.jsx`**
```jsx
// null 체크로 변경 (""은 드라이브 목록 복귀 센티넬이므로 표시 필요)
{parentPath !== null && (
  <button onClick={() => navigate(parentPath)}>
    ← {parentPath === '' ? '드라이브 목록' : '상위 폴더'}
  </button>
)}
```

**`FileCollectorService.java`**
```java
// 파일명만 추출하여 dot 검색 (경로 폴더명의 dot 오탐 방지)
private String extractExtension(String pathStr) {
    if (pathStr == null) return null;
    java.nio.file.Path p = Paths.get(pathStr);
    String fileName = p.getFileName() != null ? p.getFileName().toString() : pathStr;
    int lastDot = fileName.lastIndexOf('.');
    if (lastDot < 0 || lastDot == fileName.length() - 1) return null;
    String ext = fileName.substring(lastDot + 1).trim().toLowerCase(Locale.ROOT);
    return ext.isEmpty() ? null : ext;
}
```

## 3. 컴퓨터 시작 시 자동 실행 설정 기능 추가

### 기존 상태
```javascript
// main.js — app.whenReady()
if (app.isPackaged && !app.getLoginItemSettings().openAtLogin) {
  app.setLoginItemSettings({ openAtLogin: true }); // 첫 실행 시 무조건 자동 등록
}
```
트레이 메뉴에만 토글이 있고, 설정 페이지에는 노출되지 않았음.

### 문제점
- 사용자가 원하지 않아도 앱 설치 즉시 시작 프로그램에 자동 등록됨
- 끄려면 트레이 메뉴를 찾아야 해서 직관적이지 않음

### 변경 내용
**`main.js`**
```javascript
// 강제 자동 등록 코드 삭제

// IPC 핸들러 추가
ipcMain.handle('get-login-item', () => app.getLoginItemSettings().openAtLogin);
ipcMain.handle('set-login-item', (_event, enabled) => {
  app.setLoginItemSettings({ openAtLogin: !!enabled });
  tray.setContextMenu(buildContextMenu()); // 트레이 메뉴도 즉시 동기화
  return app.getLoginItemSettings().openAtLogin;
});
```

**`preload.js`**
```javascript
getLoginItem: () => ipcRenderer.invoke('get-login-item'),
setLoginItem: (enabled) => ipcRenderer.invoke('set-login-item', enabled),
```

**`SettingNotifyPage.jsx`**
- 페이지 진입 시 `getLoginItem()`으로 현재 값 로드
- "컴퓨터 시작 시 자동 실행" 토글 추가 (Electron 환경에서만 표시)
- 토글 변경 즉시 `setLoginItem(enabled)` 호출 (저장 버튼 불필요)
