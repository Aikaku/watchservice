## 1. 알림 심각도별 소리 알림 구현 (#27)

### 기존 상태
소리 알림 기능 없음. 새 알림 발생 시 시각적 표시만 존재.

### 문제점
DANGER/WARNING 같은 긴급 알림이 발생해도 사용자가 화면을 보고 있지 않으면 인지 불가.

### 변경 내용
- `AudioAlert.jsx` 신규 생성 (`watchservice_fe/src/components/common/`)
  - Web Audio API (파일 없음, 브라우저 내장) 로 심각도별 소리 생성
  - DANGER: sawtooth 880Hz 삼중 비프 / WARNING: square 660Hz 이중 비프 / SAFE: sine 523Hz 단음
  - 10초 폴링, 최신 알림 ID 변화 시에만 재생 (처음 로드 시는 기준 ID만 기록)
- `MainLayout.jsx`에 `<AudioAlert />` 마운트

## 3. 오탐 신고 구현 (#23)

### 기존 상태
AI가 랜섬웨어로 판정한 알림을 오탐으로 표시할 방법 없음.

### 문제점
정상 활동이 DANGER/WARNING으로 분류될 때 사용자가 피드백을 줄 수 없어 재탐지 시에도 반복 알림 발생.

### 변경 내용
- **백엔드**
  - `notification` 테이블에 `is_false_positive INTEGER DEFAULT 0` 컬럼 추가 (마이그레이션 safe)
  - `NotificationRepository.java`: SELECT 쿼리에 `is_false_positive` 포함, `notificationRowMapper()` 업데이트, `markFalsePositive(ownerKey, id)` 추가
  - `Notification.java`: `falsePositive` 필드 추가
  - `NotificationResponse.java`: `falsePositive` 필드 추가
  - `NotificationService.java`: `markFalsePositive(ownerKey, id)` 추가
  - `NotificationController.java`: `PATCH /notifications/{id}/false-positive` 엔드포인트 추가
- **프론트엔드**
  - `HttpClient.js`: `patch()` 함수 추가
  - `NotificationsApi.js`: `reportFalsePositive(id)` 추가
  - `NotificationDetailPage.jsx`: "오탐 신고" 버튼 추가 (이미 신고된 경우 비활성, 완료 시 "오탐 신고 완료" 표시)

## 4. 파일 확장자 분포 차트 구현 (#24)

### 기존 상태
어떤 확장자의 파일이 얼마나 활동하는지 확인할 방법 없음.

### 문제점
랜섬웨어는 특정 확장자를 집중 공격하는 경향이 있어, 확장자 분포를 시각화하면 이상 징후 파악에 유용함.

### 변경 내용
- **백엔드**
  - `LogRepository.java`: `ExtensionStatRow` record + `findExtensionStats(ownerKey, limit)` 추가
    - `COALESCE(ext_after, ext_before, '(없음)')` 로 null 처리, `GROUP BY ext ORDER BY cnt DESC`
  - `LogService.java`: `getExtensionStats(ownerKey, limit)` 추가
  - `LogController.java`: `GET /logs/extension-stats?limit=20` 엔드포인트 추가
- **프론트엔드**
  - `LogsApi.js`: `fetchExtensionStats(limit)` 추가
  - `ExtensionStatsPage.jsx` 신규 생성 (`/logs/extension-stats` 라우트)
    - 20가지 색상을 순환하는 가로 막대 차트 (커스텀 SVG 없이 div 스타일링)
    - 총 이벤트 수 및 비율(%) 표시
  - `App.js`: 라우트 추가
  - `LogsPage.jsx`: "확장자 분포" 버튼 추가
