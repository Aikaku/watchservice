## 1. 배포 사전작업 완료 (AI 서버 Railway 배포 + 인스톨러 빌드)

### 기존 상태
- `application.yml`의 AI 서버 URL 기본값이 `http://localhost:8001`로 되어 있어, 사용자가 설치하면 AI 서버에 연결 불가
- 관리자 비밀번호 기본값이 평문 `123456789`라 로그인 차단됨
- `jre_build.sh`의 JAVA_HOME 자동 탐지 로직이 macOS에서 `/usr/bin/java`(시스템 symlink)를 잡아 `/usr/jmods`를 찾아 실패

### 문제점
사용자 PC에 Python이 없으므로 AI 서버를 클라우드에 배포하고, 그 URL을 빌드 시점에 기본값으로 박아넣어야 한다. 그렇지 않으면 설치 즉시 탐지 기능이 동작하지 않는다.

### 변경 내용

**1) AI 서버 Railway 배포**
- `~/watchservice-ai-server/` 레포 구성: `api_server.py` + `artifacts/` + `configs/` + `requirements.txt` + `Procfile` + `railway.toml`
- `requirements.txt`에 `scikit-learn==1.6.1` 누락으로 첫 배포 실패 → 추가 후 재배포 성공
- 배포 URL: `https://web-production-728bf.up.railway.app`
- 헬스체크 확인: `{"ok":true,"model_loaded":true,"feature_count":9,...}`

**2) `application.yml` AI 서버 URL 기본값 교체**
```yaml
# 변경 전
ai:
  analyze:
    url: ${AI_ANALYZE_URL:http://localhost:8001/api/analyze}
  family:
    url: ${AI_FAMILY_URL:http://localhost:8001/predict}

# 변경 후
ai:
  analyze:
    url: ${AI_ANALYZE_URL:https://web-production-728bf.up.railway.app/api/analyze}
  family:
    url: ${AI_FAMILY_URL:https://web-production-728bf.up.railway.app/predict}
```

**3) `application.yml` 관리자 비밀번호 BCrypt 해시로 교체**
```yaml
# 변경 전
admin:
  password: ${ADMIN_PASSWORD:123456789}

# 변경 후
admin:
  password: ${ADMIN_PASSWORD:$2b$12$jw2C0RVKMgk9qj.1v6nm0O81arkMq4Vbqst4MwR0I/e1GgMOCsOMC}
```
비밀번호 `135086`에 해당하는 BCrypt 해시. 환경변수 미설정 시에도 로그인 가능.

**4) `jre_build.sh` JAVA_HOME 자동 탐지 수정**
```bash
# 변경 전: which java → symlink 해소 → JAVA_HOME=/usr (잘못된 경로)

# 변경 후: macOS에서 /usr/libexec/java_home -v 17 우선 사용
if [ "$(uname)" = "Darwin" ] && [ -x "/usr/libexec/java_home" ]; then
  JAVA_HOME=$(/usr/libexec/java_home -v 17 2>/dev/null \
    || /usr/libexec/java_home 2>/dev/null || true)
fi
```
