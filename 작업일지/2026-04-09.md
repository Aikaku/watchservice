## 1. [추가 기능] 소리 알림 ON/OFF 연동

### 기존 상태
```js
// AudioAlert.jsx — check() 내부
if (latest.aiLabel !== lastLabel.current) {
  playAlert(latest.aiLabel); // 항상 재생
}
```
`SettingNotifyPage`에서 소리 설정을 저장해도 `AudioAlert.jsx`는 무시하고 항상 재생.

### 문제점
사용자가 소리 알림을 끄면 조용해져야 하는데 설정과 관계없이 계속 경보음이 울림.

### 변경 내용
```js
function isSoundEnabled() {
  try {
    const raw = localStorage.getItem('watchservice.notifySettings');
    if (!raw) return true;
    return JSON.parse(raw).sound !== false;
  } catch { return true; }
}

// check() 내부
if (isSoundEnabled()) playAlert(latest.aiLabel);
```
`SettingNotifyPage`가 `localStorage`에 쓰는 `sound` 플래그를 읽어 ON/OFF 연동.

## 5. [추가 기능] 감시 폴더 삭제·이름 변경 감지

### 기존 상태
NIO `WatchService`는 등록된 디렉토리 *내부* 이벤트만 수신. 등록된 루트 폴더 자체가 삭제되면 이벤트를 받을 수 없음.

### 문제점
랜섬웨어가 감시 대상 폴더 전체를 삭제·이름 변경해도 탐지하지 못하고 감시가 자동 중단됨.

### 변경 내용
`WatcherService.java`에 상위 디렉토리 감시 추가:
```java
// 각 루트 폴더의 부모 디렉토리도 WatchKey 등록
state.parentKeyToRoots.put(parentKey, List.of(root));

// watchLoop — 부모 키에서 DELETE 이벤트 감지 시
if (state.parentKeyToRoots.containsKey(key)) {
    // 루트 폴더가 삭제됐으면 즉시 DANGER
    triggerRootDeletedDanger(ownerKey, deletedPath);
}

private void triggerRootDeletedDanger(String ownerKey, Path path) {
    Notification n = Notification.builder()
        .aiLabel("DANGER").aiScore(1.0).build();
    notificationService.saveNotification(n);
}
```
AI 서버를 거치지 않고 `NotificationService.saveNotification()` 직접 호출로 즉시 DANGER 발생.

## 7. [상용화 검토] UserAuthService BCrypt 적용

### 기존 상태
```java
// UserAuthService.java
return userPassword.equals(password); // 평문 비교
```
`AdminAuthService`는 BCrypt를 강제하지만 일반 사용자 인증은 환경변수의 비밀번호를 평문으로 비교.

### 문제점
환경변수에 비밀번호가 평문으로 저장되고 평문 비교되므로 보안 일관성이 없음.

### 변경 내용
```java
private static final BCryptPasswordEncoder ENCODER = new BCryptPasswordEncoder();

public boolean authenticate(String password) {
    if (!authEnabled) return true;
    if (password == null || password.isBlank()) return false;
    // BCrypt 해시인 경우 BCrypt 비교, 평문인 경우 직접 비교 (하위 호환)
    if (userPassword.startsWith("$2a$") || userPassword.startsWith("$2b$")) {
        return ENCODER.matches(password, userPassword);
    }
    return userPassword.equals(password);
}
```
`USER_PASSWORD`를 BCrypt 해시로 설정하면 BCrypt 검증, 평문이면 기존 동작 유지(경고 로그 출력). `@PostConstruct`에서 평문 설정 시 경고 로그 추가.

## 9. [코드 검토] application.yml YAML 구조 수정

### 기존 상태
```yaml
management:
  endpoints:
    ...
  endpoint:
    health:
      show-details: never

  datasource:          # 2칸 들여쓰기 → management.datasource.url 로 파싱됨
    url: jdbc:sqlite:log.db
  sql:
    init:
      mode: never

spring:                # 두 번째 spring: 블록 — 메일 설정만 포함
  mail:
    host: ${SMTP_HOST:smtp.gmail.com}
```

### 문제점
`datasource:`, `sql:` 블록이 `management:` 아래 2칸 들여쓰기로 배치되어 SnakeYAML이 `management.datasource.url`로 파싱. `spring.datasource.url`이 설정되지 않으면 DataSource 자동 구성 실패 → `JdbcTemplate` 빈 생성 불가 → 서버 시작 실패 가능. `spring:` 키가 두 번 등장하는 구조도 명확성을 해침.

### 변경 내용
`datasource:`, `sql:`, `mail:` 블록을 모두 `spring:` 최상위 블록 아래로 이동. `spring:` 키가 파일에서 한 번만 등장하도록 구조 통일.
```yaml
spring:
  application:
    name: watchservice-agent
  main:
    headless: true
  datasource:          # ← spring 아래로 이동
    url: jdbc:sqlite:log.db
  sql:
    init:
      mode: never
  mail:                # ← spring 아래로 통합 (두 번째 spring: 블록 제거)
    host: ${SMTP_HOST:smtp.gmail.com}
```

## 10. [코드 검토] AdminAuthInterceptor 응답 형식 통일

### 기존 상태
```java
// AdminAuthInterceptor.java:52
response.getWriter().write("{\"error\":\"admin_auth_required\"}");
```

### 문제점
나머지 모든 API 응답은 `{success, data, message}` 형식(`ApiResponse`)을 사용하는데, 관리자 인증 401 응답만 `{error}` 형식으로 달라 프론트에서 일관성 없는 에러 파싱 발생.

### 변경 내용
```java
response.getWriter().write("{\"success\":false,\"data\":null,\"message\":\"admin_auth_required\"}");
```
`ApiResponse`의 `{success, data, message}` 형식과 동일하게 맞춤.
