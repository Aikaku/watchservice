## 1. [추가 기능] 소리 알림 ON/OFF 연동

### 기존 상태
```js
// AudioAlert.jsx — check() 내부
if (latest.aiLabel !== lastLabel.current) {
  playAlert(latest.aiLabel); // 항상 재생
}
```
`SettingNotifyPage`에서 소리 설정을 저장해도 `AudioAlert.jsx`는 무시하고 항상 재생.

### 문제점
사용자가 소리 알림을 끄면 조용해져야 하는데 설정과 관계없이 계속 경보음이 울림.

### 변경 내용
```js
function isSoundEnabled() {
  try {
    const raw = localStorage.getItem('watchservice.notifySettings');
    if (!raw) return true;
    return JSON.parse(raw).sound !== false;
  } catch { return true; }
}

// check() 내부
if (isSoundEnabled()) playAlert(latest.aiLabel);
```
`SettingNotifyPage`가 `localStorage`에 쓰는 `sound` 플래그를 읽어 ON/OFF 연동.

if (data.status === 'DONE') {

{scanResult && <ScanResultModal result={scanResult} onClose={() => setScanResult(null)} />}

```js
// 3초 간격 폴링 — 새 DANGER id 감지 시
function showEmergencyWindow(notification) {
  emergencyWindow = new BrowserWindow({ fullscreen: true, alwaysOnTop: true, ... });
  emergencyWindow.loadFile('assets/emergency.html', { query: { ...params } });
}
```
`assets/emergency.html` 신규 생성: 붉은 배경, 탐지 정보 표시, Web Audio API 경보음(3톤), "확인" 버튼으로 닫기.

## 5. [추가 기능] 감시 폴더 삭제·이름 변경 감지

### 기존 상태
NIO `WatchService`는 등록된 디렉토리 *내부* 이벤트만 수신. 등록된 루트 폴더 자체가 삭제되면 이벤트를 받을 수 없음.

### 문제점
랜섬웨어가 감시 대상 폴더 전체를 삭제·이름 변경해도 탐지하지 못하고 감시가 자동 중단됨.

### 변경 내용
`WatcherService.java`에 상위 디렉토리 감시 추가:
```java
// 각 루트 폴더의 부모 디렉토리도 WatchKey 등록
state.parentKeyToRoots.put(parentKey, List.of(root));

// watchLoop — 부모 키에서 DELETE 이벤트 감지 시
if (state.parentKeyToRoots.containsKey(key)) {
    // 루트 폴더가 삭제됐으면 즉시 DANGER
    triggerRootDeletedDanger(ownerKey, deletedPath);
}

private void triggerRootDeletedDanger(String ownerKey, Path path) {
    Notification n = Notification.builder()
        .aiLabel("DANGER").aiScore(1.0).build();
    notificationService.saveNotification(n);
}
```
AI 서버를 거치지 않고 `NotificationService.saveNotification()` 직접 호출로 즉시 DANGER 발생.

## 6. [추가 기능] 이메일 알림

### 기존 상태
DANGER 탐지를 PC 화면으로만 확인 가능. 외부에서 인지할 방법 없음.

### 문제점
PC를 보지 않는 상황(자리 비움, 밤 등)에서 랜섬웨어가 탐지돼도 즉시 인지하지 못함.

### 변경 내용
**백엔드:**
- `build.gradle`: `spring-boot-starter-mail` 추가
- `application.yml`: `spring.mail` SMTP 설정 블록 추가 (환경변수: `SMTP_HOST`, `SMTP_PORT`, `SMTP_USERNAME`, `SMTP_PASSWORD`)
- `EmailNotificationService.java` 신규: `@Async` HTML 이메일 발송, 테스트 발송 지원
- `SettingsRepository.java` 수정: `app_settings (owner_key, key, value)` 테이블 추가, `getAppSetting`/`setAppSetting` 추가
- `SettingsService.java` 수정: `getAlertEmail`/`updateAlertEmail` 추가
- `SettingsController.java` 수정: `GET/PUT /settings/alert-email`, `POST /settings/alert-email/test` 추가
- `NotificationService.java` 수정: `saveNotification()` 내 DANGER 시 `EmailNotificationService.sendDangerAlert()` 호출
- `WatchserviceAgentApplication.java` 수정: `@EnableAsync` 추가

**프론트엔드:**
- `SettingApi.js` 수정: `fetchAlertEmail`, `updateAlertEmail`, `sendTestEmail` 추가
- `SettingEmailPage.jsx` 신규: 수신 이메일 입력, 저장, 테스트 발송 버튼
- `App.js` 수정: `/settings/email` 라우트 추가
- `SettingHomePage.jsx` 수정: "이메일 알림" 메뉴 카드 추가
