## 1. 오탐 신고 프론트엔드 완성

### 기존 상태
백엔드 `PATCH /notifications/{id}/false-positive`와 `is_false_positive` 컬럼은 구현됐으나, 프론트엔드에서 두 가지가 미구현 상태였다.
- 알림 목록에서 오탐 신고된 항목을 구별할 수 없음
- 신고 시 영향 받은 파일 경로가 예외 규칙에 자동 등록되지 않음

### 문제점
오탐 알림이 목록에 정상 알림과 동일하게 표시되어 반복 노이즈가 발생하고, 신고 후에도 동일 경로가 다시 탐지됐다.

### 변경 내용
**`NotificationPage.jsx`**
- `item.falsePositive`가 true이면 `opacity: 0.55`, 제목에 `textDecoration: 'line-through'` 적용
- `[오탐]` 배지 추가

**`NotificationDetailPage.jsx`**
- `reportFalsePositive()` 성공 후 `affectedPaths`(최대 10개)를 `createExceptionRule({ type: 'PATH', ... })`로 자동 등록
- 등록 성공 개수를 초록색 문구로 표시

## 2. 탐지 리포트 PDF 생성

### 기존 상태
탐지 내역을 문서로 출력할 방법이 없었다.

### 문제점
탐지 내역을 외부에 보고하거나 증거로 남길 수 없음.

### 변경 내용
**백엔드 (`build.gradle`)**
- `org.apache.pdfbox:pdfbox:3.0.3` 의존성 추가

**`report/ReportService.java`** (신규)
- 1페이지: 기간·생성시각·요약통계(총계/DANGER/WARNING/SAFE)
- 2페이지~: 탐지 목록 테이블 (시각, 위험도, AI점수, 패밀리, 파일수, 오탐여부)
- 최대 1000건 조회, 페이지 자동 넘김

**`report/ReportController.java`** (신규)
- `GET /api/report?from=&to=` — PDF를 `Content-Disposition: attachment`로 반환

**`SettingApi.js`** — `buildReportUrl()` 추가

**`NotificationStatsPage.jsx`** — 하단에 날짜 범위 입력 + PDF 다운로드 `<a>` 버튼 추가

## 3. 감시 스케줄 설정

### 기존 상태
감시가 항상 켜져 있어 백업·배치 작업 시간대에도 오탐이 발생했다.

### 문제점
특정 요일/시간대에만 감시를 제한할 방법이 없었음.

### 변경 내용
**`SettingsService.java`** — `getWatchSchedule()` / `updateWatchSchedule()` 추가 (`app_settings` 테이블 재활용)

**`SettingsController.java`** — `GET /settings/schedule`, `PUT /settings/schedule` 추가

**`WatcherService.java`**
- `pauseWatcher(ownerKey)`: `pausedRoots` 맵에 경로를 보존 후 `stopWatching()` 호출
- `resumeWatcher(ownerKey)`: `pausedRoots` 경로로 `startWatchingMultiple()` 호출

**`WatchSchedulerService.java`** (신규)
- `@Scheduled(fixedRate = 60_000)` — 매 1분 체크
- 스케줄 JSON: `{"enabled":true,"days":[1,2,3,4,5],"startTime":"09:00","endTime":"18:00"}`
- `days`는 ISO-8601 요일 (1=월 … 7=일)

**`WatchserviceAgentApplication.java`** — `@EnableScheduling` 추가

**`SettingApi.js`** — `fetchWatchSchedule()` / `updateWatchSchedule()` 추가

**`SettingSchedulePage.jsx`** (신규) — 요일 토글 버튼, 시간 범위 입력, 저장 버튼

**`App.js`** — `/settings/schedule` 라우트 추가

**`SettingHomePage.jsx`** — "감시 스케줄" 메뉴 카드 추가

## 4. 파일 권한 보안 감사

### 기존 상태
감시 폴더 내 파일에 불필요한 others 권한이 있어도 탐지할 방법이 없었다.

### 문제점
불필요한 파일 권한 방치 시 다른 사용자 계정에서 파일 접근·변조 가능.

### 변경 내용
**`build.gradle`** — `net.java.dev.jna:jna-platform:5.14.0` 추가

**`audit/AuditResult.java`** (신규) — 감사 결과 엔티티

**`audit/AuditRepository.java`** (신규)
- `file_audit` 테이블 (id, owner_key, file_path, permissions, issue, audited_at)
- `saveAll()`: ownerKey 기존 결과 삭제 후 일괄 저장 (최신 1회분만 유지)
- `findByOwner()`: 최신 감사 결과 조회

**`audit/FilePermissionAuditService.java`** (신규)
- `@Scheduled(fixedRate = 6h)` — 매 6시간 자동 스캔
- macOS/Linux: `Files.getPosixFilePermissions()` — others-write / others-execute 탐지
- Windows: POSIX 미지원 안내 메시지만 기록
- `runAuditForOwner()` — 수동 즉시 실행 지원

**`audit/AuditController.java`** (신규)
- `GET /api/audit` — 최신 감사 결과 + 마지막 실행 시각
- `POST /api/audit/run` — 수동 즉시 감사

**`SettingApi.js`** — `fetchAuditResults()` / `runAudit()` 추가

**`SettingAuditPage.jsx`** (신규) — 결과 테이블, 지금 감사 실행 버튼, 마지막 감사 시각

**`App.js`** — `/settings/audit` 라우트 추가

**`SettingHomePage.jsx`** — "파일 권한 감사" 메뉴 카드 추가
