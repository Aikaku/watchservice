## Slide 1 — Title (30초)

> "Hello, we are Team [팀명]. Today we present **WatchService Agent**, a real-time ransomware detection system powered by AI behavior analysis."

> "Our system monitors file activity in real time, uses an XGBoost model to detect ransomware behavior, and immediately alerts users when danger is detected."

---

## Slide 2 — System Overview (1분)

> "Ransomware is a critical threat — it encrypts your files and demands ransom. Traditional antivirus tools rely on known signatures, so they often miss new variants."

> "Our approach is different. Instead of scanning file content, we monitor **how files behave**. No matter how a ransomware variant is disguised, it always has to do one thing: rapidly encrypt large numbers of files."

> "WatchService Agent detects this behavior in real time and alerts the user before widespread damage occurs."

---

## Slide 3 — Key Features (1분)

> "The system has six core features."

> "First, **real-time file monitoring** using Java NIO WatchService — it watches selected folders recursively and captures every create, modify, and delete event."

> "Second, **AI-powered detection**. Every 3 seconds, we aggregate events into 9 behavioral feature vectors and feed them into an XGBoost classifier."

> "A key feature is **entropy analysis**. Entropy measures how random file content is. Normal files have low entropy; encrypted files have very high entropy. When we see a sudden entropy spike, that's a strong signal of ransomware."

> "When DANGER is detected, the system triggers an **instant alert** — a fullscreen popup, an email notification, and a Gemini AI-generated response guide."

---

## Slide 4 — Technical Architecture (1분)

> "The architecture consists of three layers."

> "The **React frontend** provides the dashboard, notification viewer, and settings."

> "The **Spring Boot backend** is the core engine. WatcherService listens for file events, EventWindowAggregator computes the 9 features every 3 seconds, and AiService calls the AI server."

> "The **Python AI server** runs FastAPI and hosts the XGBoost model. It receives the 9 features and returns a SAFE, WARNING, or DANGER verdict with a confidence score."

> "When DANGER is detected, the backend also calls Gemini API to generate a natural-language response guide tailored to the specific threat."

## Demo Script (약 5분)

### Step 1 — 앱 실행 & 감시 폴더 설정 (45초)
- WatchService Agent 실행
- 대시보드 화면 보여주기 (SAFE 상태, 초록색)
- Settings → Watch Folders → 테스트 폴더 추가
- "The app is now actively monitoring this folder."

### Step 2 — 이메일 알림 설정 (30초)
- Settings → Alert Email → 이메일 입력
- 테스트 발송 버튼 클릭
- 수신함에서 테스트 메일 확인
- "Email alerts are configured and working."

### Step 3 — 정상 활동 시연 (30초)
- 텍스트 파일 2~3개 생성/수정/삭제
- 대시보드가 SAFE를 유지하는 것 보여주기
- "Normal file activity — the system correctly reports SAFE."

### Step 4 — 랜섬웨어 행동 시뮬레이션 (1분)
- 시뮬레이션 스크립트 실행 (파일 20개 이상 급속 암호화 모방)
- "Watch carefully — I'm now simulating ransomware behavior: mass file renaming and high-entropy overwrites."

### Step 5 — DANGER 탐지 (1분)
- 전체화면 긴급 경보창 등장
- 이메일 수신 확인
- "The system detected DANGER and immediately triggered a fullscreen alert and sent an email notification."

### Step 6 — 대응 가이드 & 대시보드 (1분)
- Gemini AI 대응 가이드 보여주기
- 대시보드 → 알림 상세 → 영향 파일 목록
- 통계 페이지 보여주기
- "The Gemini AI generated a specific response guide. The dashboard shows the full detection history and affected files."
