# WatchService Agent — 인수테스트 시나리오

> 이 문서는 실제 사용자가 앱을 설치한 후 주요 기능을 직접 사용해보며 정상 동작 여부를 확인하기 위한 가이드입니다.  
> 기술적 배경 없이도 순서대로 따라할 수 있도록 작성되었습니다.

---

## 다운로드

**https://aikaku.github.io/watchservice/download_site/**

---

## 테스트 전 준비사항

- WatchService Agent 설치 완료 (`.dmg` 또는 `.msi` 실행 후 설치)
- 인터넷 연결 (AI 분석 서버 통신용)
- 테스트용 빈 폴더 하나 준비 (예: 바탕화면에 `test_folder` 생성)

## S-01 — 앱 설치 및 최초 실행

| 항목 | 내용 |
|------|------|
| **목적** | 인스톨러로 앱을 설치하고, 대시보드가 정상적으로 열리는지 확인한다 |
| **액터** | 일반 사용자 |
| **사전 조건** | 인스톨러 파일(`.dmg` 또는 `.msi`)이 다운로드되어 있다 |
| **사후 조건** | 대시보드가 열리고 시스템 트레이에 앱 아이콘이 등록된다 |

**시나리오 흐름**

| 단계 | Actor Action | System Response |
|------|-------------|----------------|
| 1 | 인스톨러 파일을 더블클릭한다 | |
| 2 | | 설치 마법사 또는 macOS 마운트 창이 열린다 |
| 3 | **(macOS)** Applications 폴더에 복사 후 앱 아이콘 **우클릭(또는 Control+클릭) → "열기"** 클릭<br>**(Windows)** SmartScreen 경고 → "추가 정보" → "실행" 클릭 | |
| 4 | | 보안 경고가 해제되고 설치가 진행된다 |
| 5 | 설치 완료 후 앱을 실행한다 | |
| 6 | | 로딩 화면이 표시되고, 내부 Spring Boot 서버 기동을 위해 헬스체크를 반복 시도한다 |
| 7 | 로딩 화면이 사라질 때까지 기다린다 | |
| 8 | | 헬스체크 성공 후 메인 대시보드가 자동으로 열린다 |
| 9 | 대시보드 상단 배지와 시스템 트레이를 확인한다 | |
| 10 | | 상단에 "감시 중지 상태" 배지가 표시되고, 트레이에 앱 아이콘이 등록된다 |

**예외 흐름**

| 단계 | 상황 | System Response |
|------|------|----------------|
| 6 | 60초 내 서버가 기동되지 않음 | 오류 화면이 표시되고 "재시도" 버튼이 나타난다 |

## S-02 — 감시 폴더 등록 및 감시 시작

| 항목 | 내용 |
|------|------|
| **목적** | 보호할 폴더를 등록하고 실시간 감시를 시작한다 |
| **액터** | 일반 사용자 |
| **사전 조건** | 앱이 실행 중이다. 바탕화면에 `test_folder` 폴더가 존재한다 |
| **사후 조건** | `test_folder`가 감시 대상으로 등록되고 실시간 감시가 활성화된다 |

**시나리오 흐름**

| 단계 | Actor Action | System Response |
|------|-------------|----------------|
| 1 | 왼쪽 메뉴에서 **설정 → 감시 폴더**를 클릭한다 | |
| 2 | | 감시 폴더 목록 페이지가 열린다 |
| 3 | "폴더 추가" 버튼을 클릭한다 | |
| 4 | | 폴더 탐색기 모달이 열린다 |
| 5 | 탐색기에서 `test_folder`를 선택하고 확인을 클릭한다 | |
| 6 | | 모달이 닫히고 감시 폴더 목록에 해당 경로가 추가된다 |
| 7 | 왼쪽 메뉴에서 **대시보드**로 이동한다 | |
| 8 | | 대시보드에 등록된 감시 폴더 목록이 표시된다 |
| 9 | "감시 시작" 버튼을 클릭한다 | |
| 10 | | 상단 배지가 **"감시 중"** 으로 변경되고, 버튼이 "감시 중지"로 바뀐다 |

**예외 흐름**

| 단계 | 상황 | System Response |
|------|------|----------------|
| 5 | 이미 등록된 폴더를 다시 추가 시도 | "이미 등록된 폴더입니다" 안내 메시지가 표시된다 |
| 9 | 등록된 감시 폴더가 없는 상태에서 감시 시작 | "감시 폴더를 먼저 등록하세요" 안내가 표시된다 |

## S-03 — 랜섬웨어 행위 탐지 (파일 대량 변경)

| 항목 | 내용 |
|------|------|
| **목적** | 짧은 시간 내 파일을 대량 생성·변경하면 시스템이 탐지하는지 확인한다 |
| **액터** | 일반 사용자 (터미널 명령어로 파일 이벤트 유발) |
| **사전 조건** | S-02 완료 후 감시가 실행 중이다 |
| **사후 조건** | 대시보드에 WARNING 또는 DANGER 알림이 생성된다 |

**시나리오 흐름**

| 단계 | Actor Action | System Response |
|------|-------------|----------------|
| 1 | 터미널(macOS) 또는 명령 프롬프트(Windows)를 열고 `test_folder`로 이동한다 | |
| 2 | | 터미널이 준비된다 |
| 3 | **(macOS)** `for i in $(seq 1 20); do dd if=/dev/urandom bs=1024 count=50 of="file_$i.txt" 2>/dev/null; done` 실행<br>**(Windows)** `for /L %i in (1,1,20) do fsutil file createnew file_%i.txt 51200` 실행 | |
| 4 | | `test_folder` 안에 파일 20개가 생성된다 |
| 5 | 명령 실행 후 5~10초 대기한다 | |
| 6 | | WatcherService가 파일 이벤트를 감지하고 3초 윈도우로 집계 후 AI 서버에 전송한다 |
| 7 | 대시보드 상단 배지를 확인한다 | |
| 8 | | 배지가 **WARNING** 또는 **DANGER** 로 변경된다 |
| 9 | 왼쪽 메뉴 알림 아이콘을 확인한다 | |
| 10 | | 알림 배지에 숫자(1 이상)가 표시된다 |

**예외 흐름**

| 단계 | 상황 | System Response |
|------|------|----------------|
| 8 | SAFE로 판정됨 | 배지가 녹색 SAFE를 유지한다. 파일 수를 50개로 늘려 재시도한다 |
| 8 | AI 서버 미연결 | 탐지 알림이 생성되지 않고 대시보드에 "AI 서버 연결 실패" 안내가 표시된다 |

## S-05 — 이메일 알림 수신 확인

| 항목 | 내용 |
|------|------|
| **목적** | DANGER 탐지 시 설정한 이메일로 경보 메일이 자동 발송되는지 확인한다 |
| **액터** | 일반 사용자 |
| **사전 조건** | 앱이 실행 중이다. 수신할 이메일 주소가 준비되어 있다 |
| **사후 조건** | 설정한 이메일 주소로 경보 메일이 수신된다 |

**이메일 설정 (최초 1회)**

| 단계 | Actor Action | System Response |
|------|-------------|----------------|
| 1 | 왼쪽 메뉴에서 **설정 → 알림 설정**을 클릭한다 | |
| 2 | | 알림 설정 페이지가 열린다 |
| 3 | "이메일 알림" 섹션에 수신할 이메일 주소를 입력하고 저장한다 | |
| 4 | | 이메일 주소가 저장되었다는 안내가 표시된다 |
| 5 | "테스트 발송" 버튼을 클릭한다 | |
| 6 | | 테스트 메일 발송을 시도하고 "발송 완료" 메시지가 표시된다 |
| 7 | 입력한 이메일의 받은편지함을 확인한다 | |
| 8 | | 테스트 메일이 수신된다 |

**본 테스트 흐름**

| 단계 | Actor Action | System Response |
|------|-------------|----------------|
| 1 | S-03 시나리오를 실행하여 DANGER 탐지를 유발한다 | |
| 2 | | DANGER 판정 직후 시스템이 설정된 이메일로 경보 메일을 자동 발송한다 |
| 3 | 설정한 이메일의 받은편지함을 확인한다 | |
| 4 | | 탐지 시간, 위험도, 영향 파일 수가 포함된 경보 메일이 수신된다 |

**예외 흐름**

| 단계 | 상황 | System Response |
|------|------|----------------|
| 6 (설정) | SMTP 설정 오류 또는 Gmail 앱 비밀번호 만료 | "메일 발송 실패" 오류 메시지가 표시된다 |
| 2 (본 테스트) | DANGER가 아닌 WARNING 판정 | 이메일이 발송되지 않는다 (DANGER 전용 기능) |

## S-06 — 긴급 전체화면 경보 확인

| 항목 | 내용 |
|------|------|

| **액터** | 일반 사용자 |
| **사전 조건** | 앱이 Electron으로 실행 중이다 |
| **사후 조건** | 전체화면 경보를 확인하고 닫을 수 있다 |

**시나리오 흐름**

| 단계 | Actor Action | System Response |
|------|-------------|----------------|

| 2 | | 앱이 백그라운드에서 실행 중인 상태가 된다 |
| 3 | S-03 시나리오를 실행하여 DANGER 탐지를 유발한다 | |
| 4 | | 3초 윈도우 집계 후 AI 서버가 DANGER를 반환한다 |

| 8 | | 빨간 배경의 전체화면 경보 팝업에 위험도, 탐지 시간, 대응 지침이 표시된다 |
| 9 | 경보 팝업의 "확인" 버튼을 클릭한다 | |
| 10 | | 팝업이 닫히고 대시보드 화면으로 돌아온다 |

**예외 흐름**

| 단계 | 상황 | System Response |
|------|------|----------------|
| 4 | WARNING으로 판정됨 | 전체화면 경보 없이 대시보드 배지와 알림 배지만 갱신된다 |

## S-07 — 즉시 검사 실행

| 항목 | 내용 |
|------|------|
| **목적** | 감시 시작 전 폴더 전체를 스캔하여 기준 상태(baseline)를 등록하고, 완료 후 자동으로 감시가 시작되는지 확인한다 |
| **액터** | 일반 사용자 |
| **사전 조건** | 감시 폴더가 1개 이상 등록되어 있다 |
| **사후 조건** | 스캔이 완료되고 자동으로 실시간 감시가 시작된다 |

**시나리오 흐름**

| 단계 | Actor Action | System Response |
|------|-------------|----------------|
| 1 | 왼쪽 메뉴에서 **대시보드**로 이동한다 | |
| 2 | | 대시보드 화면이 표시된다 |
| 3 | **"즉시 검사"** 버튼을 클릭한다 | |
| 4 | | 스캔이 시작되고 진행률 바(퍼센트)가 실시간으로 업데이트된다 |
| 5 | 진행 중 **"일시 중지"** 버튼을 클릭한다 | |
| 6 | | 스캔이 일시 중지된다 |
| 7 | 다시 **"즉시 검사"** 버튼을 클릭한다 | |
| 8 | | 스캔이 재개되어 진행률이 계속 올라간다 |
| 9 | 스캔 완료 모달을 확인한다 | |
| 10 | | "스캔 완료" 모달이 표시되며 검사한 파일 수와 전체 파일 수가 표시된다. 감시가 자동으로 시작된다 |

**예외 흐름**

| 단계 | 상황 | System Response |
|------|------|----------------|
| 3 | 등록된 감시 폴더가 없는 상태에서 클릭 | "검사할 폴더가 없습니다. 먼저 폴더 추가에서 경로를 등록해주세요." 경고가 표시된다 |

| **사전 조건** | 앱이 실행 중이다 |
| **사후 조건** | 등록한 경로 또는 확장자의 파일 이벤트가 탐지 파이프라인에서 제외된다 |

**시나리오 흐름**

| 단계 | Actor Action | System Response |

| 3 | 내용 하단에 `## 테스트 문단` 한 줄을 추가한다 | |
| 4 | **저장** 버튼을 클릭한다 | |
| 5 | | "저장되었습니다" 안내가 표시된다 |
| 6 | 사이드바에서 **가이드** 메뉴를 클릭하여 사용자 가이드 페이지로 이동한다 | |
| 7 | | 방금 추가한 "테스트 문단" 항목이 가이드 페이지에 표시된다 |

**시나리오 흐름**

| 단계 | Actor Action | System Response |
|------|-------------|----------------|
| 1 | 관리자 메인에서 **로그 관리**를 클릭한다 | |
| 2 | | `/admin/logs` 페이지가 열리고 전체 세션의 이벤트 로그 목록이 표시된다 |
| 3 | "CSV 내보내기" 또는 "JSON 내보내기" 버튼을 클릭한다 | |
| 4 | | 해당 형식의 파일이 다운로드된다 |
| 5 | 다운로드된 파일을 열어 로그 항목이 포함되어 있는지 확인한다 | |
| 6 | | 이벤트 유형, 파일 경로, 타임스탬프 등이 포함된 데이터가 확인된다 |
