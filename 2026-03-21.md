## 1. CLAUDE.md 규칙 수정 — `코드/` 폴더 읽기 전용 범위 변경

### 기존 상태
```
코드/ 폴더 내 파일 수정 절대 금지 (코드/api_server.py 제외 — 편집 허용)
```
`api_server.py`만 편집 허용하고 나머지는 읽기 전용이었다.

### 문제점
규칙이 반대로 적용되어 있었다. `api_server.py`는 직접 수정 대상이 아닌 분석·개선 대상이며, 나머지 파일들도 참조용으로만 사용하면 충분하다.

### 변경 내용
`api_server.py`를 포함한 `코드/` 폴더 전체를 읽기 전용으로 통일. `api_server.py` 편집 허용 예외 조항 삭제.

**변경 파일**: `CLAUDE.md`

## 2. FEATURE_CHANGES.md — 피처 계산 개선 내역 추가 (개선 6~9)

### 기존 상태
개선 1~5 (3구간 샘플링, rename 탐지, encrypt_like 절대 엔트로피, entropy_diff_mean 필터, random_extension_flag 균일 패턴)만 문서화되어 있었다.

### 문제점
2026-03-11에 반영된 4가지 추가 개선이 문서에 누락되어 있어 변경 이력 추적이 불가능했다.

### 변경 내용
4개 섹션 신규 추가:
- **개선 6** — whitelist 확장 14 → 33개 (`.json`, `.html`, `.java` 등 일반 파일 오탐 방지)
- **개선 7** — 대용량 CREATE(≥4KB)를 writeCount로 카운트 (암호화된 복사본 생성 탐지)
- **개선 8** — 스트림 암호 in-place 패턴 탐지 (엔트로피↑·크기불변 MODIFY → 동일 경로 DELETE)
- **개선 9** — DELETE 이벤트를 touch session에 포함 (파일 접근 과소 추정 개선)

개요의 "5가지 개선" → "8가지 개선"으로 수정.

**변경 파일**: `FEATURE_CHANGES.md`

## 4. 상용화_TODO.md — P0~P4 우선순위 체계 → 카테고리별 재구성

### 기존 상태
```

## P1 — 단기 수정
## P2 — 중기 수정
## P3 — 장기 수정

```
완료 항목은 상단에, 미완료 항목은 P4에 분리되어 있었다.

### 문제점
완료·미완료가 다른 섹션에 분리되어 있어 특정 카테고리(보안, 프론트엔드 등)의 전체 진행 상황을 한눈에 볼 수 없었다. 신규 발굴 항목(P4)과 기존 항목(P0~P3)의 체계도 달라 일관성이 없었다.

### 변경 내용
보안 / 백엔드 / 프론트엔드 / AI 서버 / 배포 인프라·환경설정 5개 카테고리로 재구성. 각 카테고리 안에 ✅ 완료 / ⬜ 미완료 항목을 함께 배치. 하단 "완료 항목 요약" 테이블 제거 (중복).

신규 발굴 항목도 함께 추가:
- 관리자 평문 비교 폴백 제거 (`AdminAuthService.java:45`)
- 관리자 기본 비밀번호 폴백 제거 (`${ADMIN_PASSWORD:123456789}`)
- `.env.production` 파일 없음
- CORS 기본값에 Vite dev 서버(`localhost:5173`) 포함
- AI 서버 CORS, score 계산, benign 판정, 모델 로드 실패 처리

## 6. api_server.py 분석 및 진행률 재검토 (코드 실제 확인)

### 기존 상태
상용화_TODO.md와 진행상황표.md의 AI 서버 완료율을 13%(1/8)로 기록하고 있었다.

### 문제점
문서 작성 시점 이후 api_server.py에 이미 반영된 개선 사항이 있었으나 확인하지 않아 진행률이 과소 기록되어 있었다.

### 변경 내용
코드를 직접 확인하여 아래 3개 항목이 이미 완료된 것을 확인:

| 항목 | 개선 내용 |
|------|----------|
| AI 서버 CORS 제한 | `allow_origins=["*"]` → `os.getenv("CORS_ALLOWED_ORIGINS", "http://localhost:8080")` |
| score 계산 오류 수정 | `ransomware_score = sum(prob for item if item.family != "benign")` — 다중 클래스 오판 해결 |
| benign 판정 로직 수정 | score 임계값 기반(≥0.70 DANGER / ≥0.50 WARNING / else SAFE)으로 변경 |

AI 서버 완료율 13%(1/8) → 50%(4/8), 전체 진행률 45%(17/38) → 53%(20/38)로 상향.

**변경 파일**: `상용화_TODO.md`, `진행상황표.md`
