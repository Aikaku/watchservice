# 추가 구현 가능 기능 후보

> 작성일: 2026-03-23 | 최종 수정: 2026-03-23
> 기준: 현재 Spring Boot + React + SQLite 스택에서 구현 가능한 기능만 포함
> 총 기능 수: 50개

---

## 🔴 높은 임팩트

### 1. 이메일 알림
**설명**: DANGER 탐지 시 설정한 이메일 주소로 경고 메일 자동 발송

**사용 흐름**:
```
랜섬웨어 탐지 → Spring Boot SMTP → 사용자 이메일 수신
```

**구현 범위**:
- `spring-boot-starter-mail` 의존성 추가
- `EmailNotificationService` 구현 (탐지 시 자동 호출)
- 설정 페이지에 이메일 주소 입력 UI 추가
- 이메일 템플릿 작성 (탐지 일시, 파일명, 위험도 포함)

**환경변수**: `SMTP_HOST`, `SMTP_PORT`, `SMTP_USERNAME`, `SMTP_PASSWORD`

---

### 2. 자동 파일 격리
**설명**: DANGER 판정 시 의심 파일을 자동으로 격리 폴더로 이동하여 랜섬웨어 확산 차단

**사용 흐름**:
```
DANGER 탐지 → 해당 파일 → /격리폴더/날짜/파일명 으로 이동
대시보드에서 격리된 파일 목록 확인 + 복원 가능
```

**구현 범위**:
- `QuarantineService` 구현 (파일 이동 + DB 기록)
- 격리 폴더 경로 설정 UI
- 격리 파일 목록 페이지 (복원 버튼 포함)
- 자동 격리 ON/OFF 토글 설정

**주의**: 정상 파일 오탐 시 복원 경로 반드시 제공해야 함

---

### 3. 수동 스캔
**설명**: 특정 폴더를 즉시 스캔하여 현재 상태를 분석하는 온디맨드 기능

**사용 흐름**:
```
폴더 선택 → 스캔 시작 버튼 → 진행률 표시 → 결과 리포트
```

**구현 범위**:
- `ScanService` 구현 (기존 `FileCollectorService` 재활용)
- 스캔 결과 임시 저장 및 조회 API
- 프론트엔드 스캔 진행률 UI (현재 `ScanApi.js` 기반 구현 시작됨)
- 스캔 결과 요약 (파일 수, 의심 파일 수, 고엔트로피 파일 목록)

### 37. 긴급 격리 모드 (Emergency Lock)
**설명**: DANGER 탐지 시 또는 수동으로 "긴급 격리" 버튼을 누르면 감시 중인 모든 폴더를 읽기 전용으로 전환하여 추가 암호화를 즉시 차단

**사용 흐름**:
```
DANGER 탐지 or 수동 버튼 클릭
→ 감시 폴더 전체 파일 권한 읽기 전용 변경 (Files.setAttribute)
→ 대시보드에 "격리 모드 활성화" 배너 표시
→ 수동으로 "격리 해제" 버튼 클릭 시 복구
```

**구현 범위**:
- `EmergencyLockService` — Java `Files.setAttribute()`로 폴더 권한 변경
- `POST /api/watcher/emergency-lock` / `DELETE /api/watcher/emergency-lock` API
- 대시보드 상단 긴급 격리 버튼 + 활성화 배너

### 38. 탐지 알림 에스컬레이션
**설명**: DANGER 알림이 N분 내에 읽음 처리되지 않으면 이메일 재발송 또는 웹훅 재호출로 반복 알림

**사용 흐름**:
```
DANGER 탐지 → 1차 알림 발송
→ 10분 후 읽음 미확인 시 → 2차 알림 발송
→ 30분 후에도 미확인 시 → 3차 (최종) 알림 발송
```

**구현 범위**:
- `@Scheduled` 기반 `EscalationCheckService` — 미읽음 DANGER 알림 주기 체크
- `notification` 테이블에 `escalation_count` 컬럼 추가
- 에스컬레이션 간격·횟수 설정 UI

### 39. 아웃바운드 웹훅 (Generic Webhook)
**설명**: 탐지 발생 시 사용자가 등록한 임의의 URL로 POST 요청 발송 — SIEM, 티켓팅 시스템, 사내 채널 등 어디든 연동 가능

**페이로드 예시**:
```json
{
  "event": "RANSOMWARE_DETECTED",
  "label": "DANGER",
  "score": 0.92,
  "family": "LockBit",
  "timestamp": "2026-03-23T14:32:00",
  "affectedFiles": 47
}
```

**구현 범위**:
- `WebhookService` — `RestTemplate`으로 등록된 URL에 POST
- `webhook` DB 테이블 (url, secret, enabled)
- 설정 페이지에 웹훅 URL 등록·테스트 UI
- HMAC-SHA256 서명 헤더 추가 (수신 측 검증용)

### 40. 감시 시스템 헬스 패널


**구현 범위**:
- `HealthStatusService` — WatcherService 상태·이벤트 통계 집계
- `GET /api/health/status` API
- 대시보드 상단 헬스 패널 컴포넌트

### 41. 탐지 억제 규칙 (Suppression)
**설명**: 특정 경로·시간대에서 발생하는 알림을 억제 — 야간 백업, 바이러스 백신 스캔 등 알려진 정상 작업으로 인한 오탐 방지

**사용 흐름**:
```

```

**구현 범위**:
- `suppression_rule` DB 테이블 (path_pattern, start_time, end_time, days_of_week)
- `EventWindowAggregator`에서 억제 규칙 체크 후 알림 건너뜀
- 억제 규칙 CRUD UI
