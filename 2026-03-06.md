### 개선 1: 엔트로피 3구간 샘플링 (`EntropyAnalyzer`)
- **문제**: 파일 앞부분 4096 bytes만 읽어 엔트로피 계산 → 중간·끝부터 암호화하는 랜섬웨어 미탐
- **해결**: `computeMultiSectionEntropy(path, 4096)` 메서드 추가
  - 파일 크기 > 4096이면 앞 40% / 중간 40% / 끝 20% 구간을 `SeekableByteChannel`로 샘플링
  - 소형 파일은 기존 순차 읽기 유지
- **FileCollectorService 수정**: 호출부를 `computeMultiSectionEntropy(path, 4096)`로 교체

### 개선 2: Rename 탐지 — ext-추가 패턴 인식 (`EventWindowAggregator`)
- **문제**: `file.doc → file.doc.wncry` 패턴(WannaCry/LockBit 전형)이 score=0으로 rename 미탐
- **해결**: `detectRenameLikeCountByScore()`에 2가지 score 조건 추가
  - `crtPath.startsWith(delPath + ".")` → +3점 (ext 추가 패턴)
  - `getFileStem(del).equalsIgnoreCase(getFileStem(crt))` → +2점 (stem 동일)
  - `getFileStem()` private 메서드 추가

### 개선 3: encrypt_like — 절대 엔트로피 탐지 (`EventWindowAggregator`)
- **문제**: before 스냅샷 없는 CREATE 파일은 encrypt_like 탐지 불가
- **해결**: 조건 B 추가: `!hasEntropyPair && entropyAfter >= 7.2 && isSuspiciousExt(extAfter)`
- **신규 설정**: `watchservice.analytics.encrypt.absolute-entropy-threshold: 7.2`

### 개선 4: entropy_diff_mean 필터 (`EventWindowAggregator`)
- **문제**: entropyDiff ≈ 0인 이벤트도 평균에 포함 → 암호화 신호 희석
- **해결**: `Math.abs(entropyDiff) > 0.05`인 경우만 평균 산입
- **신규 설정**: `watchservice.analytics.entropy.significant-diff-threshold: 0.05`
- **주의**: 평균값 상승 가능 → 배포 후 모니터링 필요 (HIGH 리스크)

### 개선 5: random_extension_flag — 균일 확장자 집단 탐지 (`EventWindowAggregator`)
- **문제**: suspicious 확장자 개수만으로 판정, 동일 확장자 집단 공격 미탐
- **해결**: `extAfterFreq` 맵으로 동일 suspicious 확장자 빈도 누적, 3개 이상이면 flag=1
- **신규 설정**: `watchservice.analytics.random-ext.uniform-ext-min-count: 3`
