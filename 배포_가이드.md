## ⚡ 배포 전 반드시 먼저 해야 할 것 (AI 서버 Railway 배포)

> 이 앱은 사용자가 다운받아 설치하는 구조다. 사용자 PC에 Python이 없으므로  
> **AI 서버는 반드시 클라우드(Railway)에 미리 올려둬야 한다.**  
> AI 서버 없이 빌드하면 탐지 알림이 전혀 생성되지 않는다.

### 0단계: 사전 준비

- GitHub 계정
- Railway 계정 ([railway.app](https://railway.app) — GitHub으로 로그인)

### 3단계: Railway에 배포

1. [railway.app](https://railway.app) 로그인
2. **New Project** → **Deploy from GitHub repo**
3. `watchservice-ai-server` 레포 선택
4. 배포 자동 시작 — 완료까지 약 2~3분 소요
5. 배포 완료 후 **Settings → Networking → Generate Domain** 클릭 → 도메인 발급
6. 발급된 URL 복사 (예: `https://watchservice-ai-server-production.up.railway.app`)

> **Railway 무료 플랜 주의**: 월 $5 크레딧 소진 후 중단됨. 안정적 운영을 위해 유료 플랜($5/월) 권장.

### 5단계: `application.yml` 기본값 수정 (빌드 전 필수)

Railway URL을 받은 뒤, 아래 두 값을 `watchservice_be/src/main/resources/application.yml`에서 수정한다.

**수정 전:**
```yaml
ai:
  analyze:
    url: ${AI_ANALYZE_URL:http://localhost:8001/api/analyze}
  family:
    url: ${AI_FAMILY_URL:http://localhost:8001/predict}
```

**수정 후 (Railway URL로 교체):**
```yaml
ai:
  analyze:
    url: ${AI_ANALYZE_URL:https://[받은URL]/api/analyze}
  family:
    url: ${AI_FAMILY_URL:https://[받은URL]/predict}
```

> 이렇게 하면 사용자가 환경변수를 따로 설정하지 않아도 앱 설치 즉시 AI 서버에 연결된다.

---

### 6단계: 관리자 비밀번호 기본값 수정 (빌드 전 필수)

`application.yml`의 admin 섹션도 수정한다.  
현재 기본값 `123456789`는 평문이라 **로그인이 차단**된다.

**수정 전:**
```yaml
admin:
  password: ${ADMIN_PASSWORD:123456789}
```

**수정 후 (BCrypt 해시로 교체):**
```yaml
admin:
  password: ${ADMIN_PASSWORD:$2b$12$jw2C0RVKMgk9qj.1v6nm0O81arkMq4Vbqst4MwR0I/e1GgMOCsOMC}
```

> 위 해시는 비밀번호 `135086`에 해당하는 BCrypt 해시다.  
> 비밀번호를 바꾸고 싶으면:
> ```bash
> python3 -c "import bcrypt; print(bcrypt.hashpw(b'새비밀번호', bcrypt.gensalt()).decode())"
> ```

### 실행해야 하는 프로세스 요약

| # | 프로세스 | 포트 | 필수 여부 | 비고 |

## 3. 방식 A: Electron 데스크탑 앱 (권장)

### 3.1 빌드

```bash
# 프로젝트 루트에서
./build.sh --electron        # macOS → .dmg 생성
build.bat --electron          # Windows → .msi 생성
```

빌드 완료 후 `watchservice_electron/dist/` 에 인스톨러 파일이 생성된다.

### 3.2 환경변수 설정 (설치 전)

Electron 앱은 시스템 환경변수를 그대로 읽는다. **설치 전** 대상 PC에서 아래를 설정한다.

**macOS / Linux** (`~/.zshenv` 또는 `~/.bashrc` 에 추가):
```bash
export ADMIN_PASSWORD='$2a$10$XXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX'  # BCrypt 해시
export GEMINI_API_KEY='AIza...'          # Gemini guidance 사용 시
export AI_ANALYZE_URL='https://your-ai-server.railway.app/api/analyze'
export AI_FAMILY_URL='https://your-ai-server.railway.app/predict'

# 이메일 알림 사용 시
export SMTP_USERNAME='your@gmail.com'
export SMTP_PASSWORD='xxxx xxxx xxxx xxxx'   # Gmail 앱 비밀번호
```

**Windows** (명령 프롬프트, 관리자 권한):
```cmd
setx ADMIN_PASSWORD "$2a$10$XXXX..."
setx GEMINI_API_KEY "AIza..."
setx AI_ANALYZE_URL "https://your-ai-server.railway.app/api/analyze"
setx AI_FAMILY_URL  "https://your-ai-server.railway.app/predict"
setx SMTP_USERNAME  "your@gmail.com"
setx SMTP_PASSWORD  "xxxxxxxxxxxxxxxxxxxx"
```

> ⚠️ `setx` 명령 후 터미널을 **재시작**해야 적용된다.

### 3.3 BCrypt 비밀번호 해시 생성 방법

```bash
# Python (설치 필요: pip install bcrypt)
python3 -c "import bcrypt; print(bcrypt.hashpw(b'원하는비밀번호', bcrypt.gensalt()).decode())"

# 출력 예: $2b$12$AbCdEf...  → 이 값을 ADMIN_PASSWORD에 설정
```

### 3.4 설치 후 접속

인스톨러 실행 → 앱 시작 → `http://localhost:8080` (내장 브라우저로 자동 오픈)

## 4. 방식 B: JAR 단독 서버

### 4.1 빌드

```bash
./build.sh        # macOS/Linux
build.bat         # Windows
```

`watchservice_be/build/libs/watchservice-agent-*.jar` 생성.

### 4.2 실행

```bash
# 기본 실행 (환경변수는 시스템에 미리 설정)
java -jar watchservice-agent-*.jar

# 또는 인라인으로 환경변수 주입
ADMIN_PASSWORD='$2a$10$XXX...' \
GEMINI_API_KEY='AIza...' \
AI_ANALYZE_URL='https://your-ai-server.railway.app/api/analyze' \
AI_FAMILY_URL='https://your-ai-server.railway.app/predict' \
java -jar watchservice-agent-*.jar
```

### 4.3 프로세스 유지 (systemd, macOS/Linux 서버)

`/etc/systemd/system/watchservice.service`:
```ini
[Unit]
Description=WatchService Agent
After=network.target

[Service]
Type=simple
User=watchservice
WorkingDirectory=/opt/watchservice
EnvironmentFile=/opt/watchservice/.env
ExecStart=/usr/bin/java -jar /opt/watchservice/watchservice-agent.jar
Restart=on-failure
RestartSec=10

[Install]
WantedBy=multi-user.target
```

`/opt/watchservice/.env` (chmod 600):
```
ADMIN_PASSWORD=$2a$10$XXX...
GEMINI_API_KEY=AIza...
AI_ANALYZE_URL=https://your-ai-server.railway.app/api/analyze
AI_FAMILY_URL=https://your-ai-server.railway.app/predict
SMTP_USERNAME=your@gmail.com
SMTP_PASSWORD=xxxx xxxx xxxx xxxx
CORS_ALLOWED_ORIGINS=http://your-domain.com
```

```bash
sudo systemctl enable watchservice
sudo systemctl start watchservice
sudo systemctl status watchservice
```

### 4.4 Nginx 리버스 프록시 (HTTPS, 다수 사용자 접근 시)

`/etc/nginx/sites-available/watchservice`:
```nginx
server {
    listen 80;
    server_name your-domain.com;
    return 301 https://$host$request_uri;
}

server {
    listen 443 ssl;
    server_name your-domain.com;

    ssl_certificate     /etc/letsencrypt/live/your-domain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/your-domain.com/privkey.pem;

    location / {
        proxy_pass         http://127.0.0.1:8080;
        proxy_set_header   Host $host;
        proxy_set_header   X-Real-IP $remote_addr;
        proxy_read_timeout 60s;
    }
}
```

## 5. AI 서버 배포 (공통)

AI 서버는 **클라우드 배포(권장)** 또는 **로컬 실행** 중 선택한다.

### 5.1 클라우드 배포 (Railway) — 권장

Railway는 배포 후 잠들지 않는 유료 플랜($5/월~) 사용 시 안정적이다.

#### 전용 Git 레포 준비

```bash
mkdir ai-server && cd ai-server
git init

# 필요 파일 복사 (통합 프로젝트 루트 기준)
cp 코드/api_server.py .
cp -r 코드/artifacts ./artifacts          # model_xgb.json, features.json, classes.json
cp -r ai_server_deploy/configs ./configs  # config.yaml
cp ai_server_deploy/requirements.txt .
cp ai_server_deploy/Procfile .
cp ai_server_deploy/railway.toml .

git add . && git commit -m "init ai server"
git remote add origin https://github.com/your-org/ai-server.git
git push -u origin main
```

최종 레포 구조:
```
ai-server/
├── api_server.py
├── requirements.txt
├── Procfile
├── railway.toml
├── configs/config.yaml
└── artifacts/
    ├── model_xgb.json    ← XGBoost 모델 (필수)
    ├── features.json     ← 피처 목록 (필수)
    └── classes.json      ← 클래스 레이블 (필수)
```

#### Railway 배포

1. [railway.app](https://railway.app) → New Project → Deploy from GitHub repo
2. 위 레포 선택
3. Variables 탭:
   ```
   CORS_ALLOWED_ORIGINS=https://your-spring-boot-domain.com
   ```
4. 배포 완료 후 생성된 URL 메모 (예: `https://ai-server-prod.railway.app`)

#### Spring Boot에 AI 서버 URL 주입

```bash
AI_ANALYZE_URL=https://ai-server-prod.railway.app/api/analyze
AI_FAMILY_URL=https://ai-server-prod.railway.app/predict
```

### 5.2 로컬 실행 (개발 / 단일 PC)

```bash
cd ai_server_deploy   # 또는 코드/ 디렉토리

# 가상환경 세팅 (최초 1회)
python3 -m venv venv
source venv/bin/activate          # Windows: venv\Scripts\activate
pip install -r requirements.txt

# 실행
OUTPUT_DIR=./artifacts python3 ../코드/api_server.py
# 또는
uvicorn api_server:app --host 0.0.0.0 --port 8001
```

#### 정상 기동 확인

```bash
curl http://localhost:8001/health
# 정상: {"ok":true,"model_loaded":true,...}
```

| `SMTP_USERNAME` | (없음) | 발신 이메일 주소 | 선택 |

| `USER_PASSWORD` | (없음) | 일반 사용자 비밀번호 | 선택 |
| `AI_CONNECT_TIMEOUT_MS` | `5000` | AI 서버 연결 타임아웃 | — |
| `AI_READ_TIMEOUT_MS` | `15000` | AI 서버 응답 타임아웃 | — |

| 환경변수 | 기본값 | 설명 |
|---------|--------|------|
| `CORS_ALLOWED_ORIGINS` | `http://localhost:8080` | Spring Boot 서버 Origin |

| `PORT` | `8001` | 서버 포트 (Railway 자동 주입) |

## 7. 운영 중 유지·관리해야 할 것

### 7.1 데이터 파일 (`log.db`)

- **위치**: Spring Boot JAR 실행 경로의 `log.db` (SQLite)
- **내용**: 알림, 이벤트 로그, 감시 폴더, 설정값 전부
- **백업**: 주기적으로 복사해둔다. Spring Boot 종료 후 복사하거나 SQLite `.backup` 명령 사용

```bash
# 백업 예
cp log.db log.db.backup-$(date +%Y%m%d)

# 또는 SQLite 온라인 백업 (실행 중에도 가능)
sqlite3 log.db ".backup log.db.backup-$(date +%Y%m%d)"
```

### 7.4 디스크 공간

`log.db`가 지속적으로 커질 수 있다. 오래된 로그를 정기적으로 정리한다.

- 관리자 페이지 `/admin/logs` → 날짜 범위로 삭제 또는 CSV 내보내기 후 삭제
- 또는 SQLite에서 직접:

```bash
sqlite3 log.db "DELETE FROM event_log WHERE created_at < strftime('%s','now','-90 days') * 1000;"
sqlite3 log.db "VACUUM;"   # 실제 파일 크기 줄이기
```

### 자주 발생하는 문제

| 증상 | 원인 | 해결 |
|------|------|------|
| 관리자 로그인 실패 (401) | `ADMIN_PASSWORD`가 BCrypt 해시가 아님 | BCrypt 해시 재생성 후 환경변수 교체 |
| 탐지 알림이 전혀 안 생김 | AI 서버가 꺼져 있거나 URL이 잘못됨 | `AI_ANALYZE_URL` 확인, AI 서버 재시작 |
| Gemini 분석 내용이 비어 있음 | `GEMINI_API_KEY` 미설정 또는 만료 | API 키 재발급 후 환경변수 교체 |
| 이메일 발송 실패 | SMTP 설정 오류 또는 Gmail 앱 비밀번호 만료 | Gmail 앱 비밀번호 재생성 |
| AI 서버 첫 요청 ~30초 지연 | Render 무료 플랜 sleep 복귀 | Railway 유료 전환 또는 로컬 실행 |
| `log.db` 잠금 오류 | 동일 JAR이 2개 이상 실행됨 | 프로세스 확인 후 중복 종료 |

### 로그 위치

- **Spring Boot**: 콘솔 출력 + `watchservice_be/logs/watchservice.log` (logback 롤링)
- **AI 서버**: 콘솔 출력 (Railway/Render는 대시보드 Logs 탭)
