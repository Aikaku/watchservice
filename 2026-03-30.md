## 4. 추가기능_후보.md 표 형식으로 전면 재정리

### 기존 상태
50개 기능이 섹션별 산문 형식(설명, 사용 흐름, 구현 범위 블록)으로 작성되어 있었다. 번호도 37~42가 높은 임팩트 섹션에 끼어 있는 등 순서가 뒤섞여 있었다.

### 문제점
- 기능이 50개나 되는데 산문 형식이라 한눈에 비교 불가
- 번호 체계가 불연속적 (11→13→24→37 등)이라 참조가 어려움
- 난이도 정보가 우선순위 추천 테이블 10개에만 존재, 나머지 40개는 불명

### 변경 내용
전체 50개를 임팩트 수준별 3개 테이블로 재정리:
- `# / 기능명 / 설명 / 핵심 구현 / 난이도` 5개 컬럼
- 번호를 임팩트 그룹 내 순차 재부여 (1~15 / 16~38 / 39~50)
- 모든 50개 항목에 난이도 추가
- 우선순위 추천 테이블에 "선정 이유" 컬럼 추가

## 2. 유스케이스.md 신규 작성

### 기존 상태
시스템의 사용자 시나리오를 정리한 문서가 존재하지 않았다.

### 문제점
- 기능 목록은 있으나 사용자 관점의 흐름(기본 흐름, 대안 흐름, 예외 흐름)이 정리되어 있지 않아 개발 및 테스트 기준으로 활용하기 어려웠다.

### 변경 내용
총 29개 유스케이스를 3개 액터(일반 사용자, 관리자, 시스템) 기준으로 정의:
- **앱**: UC-APP01~02 (Electron 설치·실행, 트레이 최소화)
- **파일 감시**: UC-W01~05 (감시 시작/중지, 즉시 검사, 폴더 추가/삭제)
- **예외 설정**: UC-E01~02 (경로/확장자 예외 추가·삭제)
- **탐지(자동)**: UC-D01 (3초 윈도우 집계 → AI 분석 → 알림 저장 전체 파이프라인)
- **대시보드**: UC-DB01
- **알림**: UC-N01~03 (목록 필터링, 상세, 통계)
- **로그**: UC-L01~04 (조회, 상세, CSV/JSON 내보내기, 삭제)
- **피드백**: UC-F01
- **관리자**: UC-A01~10 (로그인/로그아웃, 세션, 피드백, 공지사항, 로그·알림 관리, 시스템 상태, 가이드 편집)

각 유스케이스마다 액터·목적·사전조건·기본흐름·대안흐름·예외흐름·사후조건 포함.
마지막에 UC 간 관계 트리와 API 호출 요약 테이블 수록.
